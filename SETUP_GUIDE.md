# GitHub Auto-Push Setup Guide
### Repo: github.com/koonergurjot/Etsy-Ai-Bot

This setup lets you edit files in a local folder using **Claude (Cowork)** or your **local LLM**, and have every change automatically pushed to GitHub — which hosts your live site via GitHub Pages.

---

## How It Works

```
You / Claude (Cowork) / Local LLM
         ↓ edit files
   Local Folder (git repo clone)
         ↓ auto_push.py detects changes
   git add → git commit → git push
         ↓
   GitHub repo (koonergurjot/Etsy-Ai-Bot)
         ↓
   GitHub Pages → Your Live Website
```

---

## Prerequisites

- **Git** — https://git-scm.com/download/win
- **Python 3.x** — https://python.org/downloads  
  *(check "Add Python to PATH" during install)*
- A GitHub account with access to `koonergurjot/Etsy-Ai-Bot`

---

## Step 1 — Clone the Repo

**Option A (easy):** Double-click `setup_repo.bat` and follow the prompts.

**Option B (manual):** Open a terminal and run:
```
git clone https://github.com/koonergurjot/Etsy-Ai-Bot C:\Projects\Etsy-Ai-Bot
```

Then copy `auto_push.py` and `start_watcher.bat` into the cloned folder.

---

## Step 2 — Connect Cowork to the Cloned Folder

1. In the Cowork app, click **Change Folder** (or the folder icon)
2. Select the folder where you cloned the repo (e.g. `C:\Projects\Etsy-Ai-Bot`)
3. Claude can now read and edit files in that folder directly

---

## Step 3 — Start the Auto-Push Watcher

Double-click **`start_watcher.bat`** inside the cloned folder.

A terminal window will open and stay running. It watches for any file changes and automatically runs:
```
git add -A
git commit -m "Auto-update: 2026-04-12 14:30:00"
git push
```

**Leave this window open** while you're working. Minimize it if you like.

---

## Step 4 — Enable GitHub Pages

1. Go to https://github.com/koonergurjot/Etsy-Ai-Bot/settings/pages
2. Under **Branch**, select `main` (or `gh-pages` if you have that branch)
3. Click **Save**
4. Your site will be live at: `https://koonergurjot.github.io/Etsy-Ai-Bot/`

---

## Step 5 — GitHub Authentication

The first time you push, Git will ask for your credentials.

**Recommended:** Use a Personal Access Token (PAT):
1. Go to https://github.com/settings/tokens
2. Click **Generate new token (classic)**
3. Give it `repo` scope
4. Copy the token — use it as your password when Git asks

To avoid being asked every time, run this once in your repo folder:
```
git config --global credential.helper store
```
Then push once and enter your username + token. Git will remember it.

## Daily Workflow

1. **Start the watcher:** Double-click `start_watcher.bat`
2. **Edit files:** Use Claude in Cowork, your local LLM, or any editor
3. **Changes auto-push** to GitHub within ~5 seconds of saving
4. **View your live site** at `https://koonergurjot.github.io/Etsy-Ai-Bot/`

## Troubleshooting

| Problem | Fix |
|---|---|
| "Not a git repository" error | Make sure you're running the watcher inside the cloned folder |
| Push fails with auth error | Set up a Personal Access Token (see Step 5) |
| Nothing pushing | Check the watcher terminal for error messages |
| Python not found | Reinstall Python and check "Add to PATH" |
